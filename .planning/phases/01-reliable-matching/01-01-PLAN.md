**Inline Levenshtein distance** (no external dep):
```typescript
function levenshtein(a: string, b: string): number {
  if (a === b) return 0;
  if (a.length === 0) return b.length;
  if (b.length === 0) return a.length;
  const dp: number[] = Array.from({ length: b.length + 1 }, (_, i) => i);
  for (let i = 1; i <= a.length; i++) {
    let prev = dp[0];
    dp[0] = i;
    for (let j = 1; j <= b.length; j++) {
      const temp = dp[j];
      dp[j] = a[i - 1] === b[j - 1]
        ? prev
        : 1 + Math.min(prev, dp[j], dp[j - 1]);
      prev = temp;
    }
  }
  return dp[b.length];
}
```

**fuzzyNameSimilarity(a: string, b: string): number**
- If both empty: return 1.0
- If one empty: return 0.0
- Apply `normalizeForFuzzy` to both
- Apply `tokenSortName` to both normalized results
- Compute `levenshtein(sortedA, sortedB)`
- Return `1 - dist / Math.max(sortedA.length, sortedB.length)` clamped to [0.0, 1.0]

**resolveOwnerKey(landTaxId: string, landNameNorm: string, reTaxId: string, reNameNorm: string): OwnerKeyResult**
- IPN-priority rule: if BOTH `landTaxId` AND `reTaxId` are non-empty strings AND they are EQUAL → return `{ matchedByIpn: true, key: landTaxId, similarity: 1.0 }`
- If BOTH are non-empty BUT NOT equal → no IPN match and name is unreliable; return `{ matchedByIpn: false, key: '', similarity: 0.0 }` (different persons identified by IPN)
- If EITHER is empty → fall back to fuzzy name: `const sim = fuzzyNameSimilarity(landNameNorm, reNameNorm); return { matchedByIpn: false, key: sim >= 0.85 ? tokenSortName(normalizeForFuzzy(landNameNorm)) : '', similarity: sim }`

**detectAreaUnit(sampleValues: number[]): 'ha' | 'm2'**
- Filter out zeros: `const vals = sampleValues.filter(v => v > 0)`
- If empty: return `'m2'`
- Compute median of vals (sort, take middle)
- If median < 50: return `'ha'` (typical land parcel in hectares is 0.1–20 ha)
- Else: return `'m2'`
